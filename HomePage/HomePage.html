1. Purpose

This project was developed to provide a way for users to learn algorithm for both reading and visualizations. This algorithm visualizer differs from other variations by allowing the user to enter their own input. By entering their own input, users will be able to get a better understanding of the limations and possibilites of the algorithms.

2. System Requirements

  1. A computer or laptop

  2. A github account

  3. An IDE

  4. The Live Server extension

3. Installation Instructions

Follow the setup guide in the Setup tab

4. Top Panel Walkthrough

Top Panel

4.1 Input Size Box and Randomize Button

Type in or use the up and down arrows on the right side of the Input Size Box to set size of the new randomized input.

Input Size Box's Up and Down Arrows

Once the input is entered, the Randomized Button will become enabled. Click the Randomize Button to create a new input list of given size. A red warning message will appear if the given input value does not meet the criteria. This criteria will differ between algorithms.

4.2 Custom User Input Box and Toggle

Type in a list of numerical values. Click on the toggle to set the current input list equal to the entered input list in the Custom User Input Box. A red warning message will appear if the entered list does not match the acceptance criteria for that algorithm. Setting the Custom User Input Toggle back to the left will set the input list back to the default input list.

4.3 Start Button

Starts the execution of the algorithm. Goes through the algorithm one frame at a time. Stops execution after the final frame.

4.4 Pause Button

Pauses the execution of the algorithm at the current frame. Only works, if the algorithm is in execution.

4.5 Reset Button

Resets the execution of the algorithm back to the first frame. If button is clicked while algorithm is in execution, then the algorithm will be paused at the current frame and reset back to the first frame. Works when algorithm is executing, paused, or completed. The button doesn't start the execution process.

4.6 Step Forward Button

Moves the algorithm's execution one frame forward. Works while the algorithm is in excution or paused but not when its completed.

4.7 Step Backward Button

Moves the algorithm's execution one fram backward. Works while the algorithm is in execution, paused, or completed but not when its on the first frame.

4.8 Progress Bar

Displays how far along the algorithm is in its execution process by having the bar get filled up with a blue bar. The user can click on different parts of the progress bar to jump to that frame in the execution. The algorithm will continue executing after this jump if it was already being executed. The bar will be completely blue once finished and completely grey at the start.

4.9 Speed Slider

Changes the speed of the algorithm's execution. Moving it to the left will slow down the execution speed and moving it to the right will increase the execution speed. The speed can be changed when algorithm is in execution or paused.

5. Left Panel Walkthrough

Left Panel

5.1 Search Bar

Click on the search bar at the top right of the left panel and type in the name of the algorithm you would like to find and a drop down menu will appear that conatains algorithms with a matching name. Once the desired algorithm appears in the drop-down menu, by clicking on said algorithm the user will be taken to said algorithm's page. The search bar can be used from any page on the application.

5.2 Algorithm Drop-Down Menus

Made up of Four Sections

  1. Brute Force: Test all possible solutions to a problem until the correct one is found.

  2. Divide and Conquer: Recursively breaks problems down into smaller subproblems, solving those subproblems, and then combing the solutions to the subproblems to solve the original problem.

  3. Dynamic Programming: Finds the shortest solution to a problem by using top-down or bottom-up approaches. Stores and resuses solutions to overlapping subproblems.

    4. Top-Down Approach: Uses recurision to break the problem down into smaller subproblems and solving those subproblems to solve the original problem.

    Bottom-Up Approach: Starts with the easiest subproblem and builds solutions to the subproblems iteratively, combining their solutions to solve the original problem.


  4. Greedy: Solves a problem by picking the best option at each step.

Click on one of the four sections and a drop-down menu will appear of the all the algorithms in that section. Each algorithm belongs to only one of the four sections.

Brute Force Drop-Down Menu

Clicking on an algorithm in the drop-down menu will go to the selected algorithm's page. Clicking on a section when its drop-down menu is open will close the drop-down menu.

6. Middle Panels Walkthrough

Quicksort's Panels

6.1 Graph Panel

Quicksort's Graph Panel

Contains a display of the given input as a bar-graph, node-graph, or matrix-graph, depending on the specific needs of each algorithm. During the execution of the algorithm, values in the graph display will become highlighted with certain colors to establish the current step in the algorithm.

6.2 Box List Panel

Quicksort's Box List Panel

Contains a panel of the input list as a box-list. During the execution of the algorithm, some of the boxes will be highlighted with certain colors to show the current step in the algorithm. For example, If it's a sorting algorithm, the displayed list will be sorted during the execution of the algorithm and eventually display the final sorted list. There are some algorithms where there will be no Box List Panel.

6.3 Step Log Panel

Quicksort's Step Log Panel

Displays in text what is happening during the current frame of the algorithm's execution. Also displays the given input at the beginning and the output at the end of the algoirthm's execution.

7. Right Panel Walkthrough

7.1 Home Page's Right Panel

Home Page's Three Tabs:

  1. User Guide: Contains an overview of the applications by describing each part of it.

  2. Setup: Contains instructions on how to setup the applicatoin.

  3. Information: Contains information on the application's contributers and source code.

7.2 Algorithm Pages' Right Panel

Algorithm Pages' Three Tabs:

  1. Description: Contains information on the algorithm such as use cases, memory usage, runtime, steps, and functionality.

  2. Pseudocode: Contains the algorithm's pseudocode.

  3. References: Contains a list of resources used to create the algorithm's descritpion and code.

FAQs

Why Does It Say Disabled For Some Of The Algorithms Custom User Input Boxes?

Disabled Customer User Input

The "Disabled" text in the Custom User Input Box means that the user cannot enter their own custom input for that algorithm. This is because those algorithms would require lengthy input with complicated instructions and a lot of restrictions. For example, Bread First Search would require the user to enter a list of adjacent nodes for each node.

Why Do Some Algorithms Have A 4th Panel?

Knapsack's 4 Panels

Some algorithms need an additional panel in order to visually explain how they operate. For example, Knapsack Problem needs a panel to show off the 2D matrix that tracks the max value for each item to get the final result, a panel for the array that contains the value of each item, a panel for the array that contains the weight of each item, and a panel for the step logger that prints out each step in the algorithm. These four panels in conjunction visually display the inner workings of the Knapsack Problem

Why Do Some Algorithms Have Only 2 Panels?

Breadth First Search's Panels

Some algorithms only have 2 panels because that is all they need to visually display their inner workings. For example, Breadth First Search only needs a panel to display how it traverses through an undirect, unweighted graph and another panel to explain each step via text.
